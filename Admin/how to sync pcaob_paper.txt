There is no matching step. The attrition chain (8,621 → 6,901 → 1,220 → 678) comes from:
  - 8,621 → 6,901: EDGAR 8-K events linked to CRSP (script 03, CIK→gvkey→PERMNO)
  - 6,901 → 1,220: Events with valid CRSP return data in the event window (script 03)
  - 1,220 → 678: apply_sample_filters() in script 05 — dropping observations missing absCar, abvol, er_pres, sic2, or any of the 6 controls, plus
  excluding financials/utilities

  So the 542-event drop isn't from propensity-score matching — it's from missing Compustat controls and polarization data, plus SIC exclusions. That
  changes the calculus.

  What this means

  The "easy win" of running an unmatched sample doesn't apply since there's no matching. The missing observations are structurally missing — no state
  mapping, no Compustat controls, or excluded industries.

  Realistic options for N=678

  1. Diagnose the biggest source of attrition — Run a quick check to see how many are lost to each filter (missing state, missing controls, SIC
  exclusions). If one category dominates, we might recover some.
  2. Relaxed controls specification — Drop one or two controls that cause the most missingness (e.g., sales_growth or btm often have spotty Compustat
  coverage). Report as a robustness column showing the main result holds with a larger N.
  3. Emphasize what's already in the paper — Permutation inference, multiple clustering, and the fact that significant results at N=678 imply large,
  detectable effect sizes. Add a sentence on ex-post power if needed.
